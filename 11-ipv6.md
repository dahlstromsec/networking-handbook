# IPv6

> **Estimated Reading Time:** 10–15 minutes  
> **Difficulty:** Beginner  
> **Prerequisites:** IPv4 Addressing

IPv6 is the successor to IPv4 and was developed to solve the growing shortage of IPv4 addresses. It introduces a much larger address space while improving scalability and supporting modern Internet growth.

Although IPv4 is still widely used, many networks today operate in **dual-stack mode**, meaning they support both IPv4 and IPv6 simultaneously.

# Why It Matters

When IPv4 was created, the Internet contained only a small number of connected devices.

Today, billions of computers, phones, servers and IoT devices require IP addresses.

IPv6 solves this limitation by providing an enormous address space capable of supporting future Internet growth.

---

# Core Concepts

Unlike IPv4, which uses **32-bit addresses**, IPv6 uses **128-bit addresses**.

Example:

```text
2001:db8:85a3::8a2e:370:7334
```

Common IPv6 address types include:

- Global Unicast
- Link-Local
- Loopback
- Multicast

One of IPv6's major advantages is that it eliminates the need for widespread NAT in many environments.

---

# How It Works

A device connects to the network.

↓

It automatically generates a Link-Local address.

↓

The router advertises IPv6 network information.

↓

The device receives a Global IPv6 address.

↓

Communication with other IPv6 networks becomes possible.

Unlike IPv4, manual configuration is often unnecessary.

# Real-World Example

Many Internet Service Providers now assign both an IPv4 address and an IPv6 address to home routers.

When visiting a website that supports IPv6, your computer may automatically use IPv6 without you noticing.

---

# Cybersecurity Context

Security professionals must understand IPv6 because:

- Modern operating systems enable IPv6 by default.
- Firewalls must filter both IPv4 and IPv6 traffic.
- Attackers may exploit poorly configured IPv6 environments.
- Packet captures often contain both IPv4 and IPv6 traffic.

Ignoring IPv6 can leave parts of a network unprotected.

# Common Mistakes

| Mistake | Explanation |
|----------|-------------|
